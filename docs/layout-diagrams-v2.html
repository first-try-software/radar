Design Principles

  1. Pages = Layout — Pages only define rows and columns, never content
  2. Partials = Content — All content comes from partials, nothing inline
  3. Shared by Default — Components live in shared/ unless page-specific
  4. Entity-Specific Presenters — Partials with conditional logic get entity-specific presenters
  5. Generic Presenters — Partials that work uniformly get generic presenters
  6. No Fixed Dimensions — Partials use min/max only, never fixed width/height
  7. Container Agnostic — Partials don't know about their layout context
  8. No Tabs — Entity lists show active items directly, archived in collapsible <details>
